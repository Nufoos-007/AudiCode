## 1. Project Overview

AudiCode is an AI-powered security vulnerability scanner designed specifically 
for "vibe-coded" (AI-generated) applications built with tools like Cursor, 
GitHub Copilot, Bolt, Lovable, and similar AI coding assistants.

Mission: Make AI-generated code secure by automating security audits with 
real-time async scanning, confidence scoring, and actionable fixes.

## 3. Architecture - Production Ready

### Current Flow:

```
┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│   Frontend   │────▶│  /api/jobs   │────▶│  Supabase   │
│  (Dashboard)│     │  (REST API)  │     │  (Jobs DB)  │
└──────────────┘     └───────────────┘     └──────────────┘
                            │
                            ▼
                     ┌───────────────┐
                     │  Background  │
                     │   Worker     │
                     └───────────────┘
                            │
                            ▼
                     ┌───────────────┐
                     │  GitHub API  │
                     │  (Scanning) │
                     └───────────────┘
                            │
                            ▼
                     ┌───────────────┐
                     │  Supabase    │
                     │  (Results)   │
                     └───────────────┘
```

### Authentication:
1. User visits https://audi-code.vercel.app
2. OAuth via Supabase (GitHub or Google)
3. User profile created with 50 free credits
4. Access dashboard with personal job history

### Async Scan Flow:
1. POST /api/jobs with repoUrl
2. Returns jobId immediately (202)
3. Background worker processes:
   - Fetches repo via GitHub API
   - Scans files (50+ patterns)
   - Checks dependencies (CVE)
   - Stores results in Supabase
4. GET /api/jobs?jobId=XXX polls for results

## 4. Database Schema (Supabase)

### Tables Created:

#### audit_jobs
```sql
id (UUID, PK)
user_id (UUID, FK to auth.users)
repo_url (TEXT)
status (queued|running|completed|failed)
progress (0-100)
score (INT)
grade (TEXT)
confidence (INT)
files_scanned (INT)
error (TEXT)
created_at (TIMESTAMP)
completed_at (TIMESTAMP)
```

#### audit_results
```sql
id (UUID, PK)
job_id (UUID, FK)
severity (critical|high|medium|low)
title (TEXT)
description (TEXT)
file (TEXT)
line (INT)
code (TEXT)
fix (TEXT)
category (TEXT)
cwe (TEXT)
owasp (TEXT)
confidence (INT)
is_false_positive (BOOLEAN)
```

#### credits_ledger
```sql
id (UUID, PK)
user_id (UUID, FK)
action (scan|fix|refund|purchase|bonus)
amount (INT)
balance (INT)
description (TEXT)
created_at (TIMESTAMP)
```

#### user_profiles
```sql
id (UUID, PK)
email (TEXT)
credits (INT, default 50)
scans_count (INT)
created_at (TIMESTAMP)
```

### Row Level Security (RLS):
- Users can only see their own jobs/results
- Service role bypass for background worker

## 5. API Endpoints

### POST /api/jobs
Create async scan job.

Request:
```json
{
  "repoUrl": "https://github.com/owner/repo",
  "token": "ghp_xxx",
  "userId": "uuid"
}
```

Response (202):
```json
{
  "jobId": "uuid",
  "status": "queued",
  "message": "Scan started"
}
```

### GET /api/jobs?jobId=xxx
Get job status and results.

Response:
```json
{
  "id": "uuid",
  "repo_url": "...",
  "status": "completed",
  "progress": 100,
  "score": 75,
  "grade": "B",
  "confidence": 85,
  "files_scanned": 42,
  "vulnerabilities": [...],
  "created_at": "...",
  "completed_at": "..."
}
```

## 6. Security Features

### Rate Limiting
- 3 scans per minute per user
- IP fallback rate limiting
- Prevents abuse and cost explosion

### Progress Tracking
- Real-time progress (0-100%)
- Stages: 10% → 30% → 50% → 70% → 85% → 100%

### Error Handling
- Failed jobs stored with error message
- Retry capability via new job creation

### Confidence Scoring (40-95)
- Based on pattern strength
- Context awareness
- File type analysis

## 7. Scanner Implementation

### Detection: 50+ OWASP Rules

| Category | Patterns |
|----------|----------|
| Secrets (A02) | OpenAI, GitHub, AWS, JWT tokens |
| Injection (A03) | SQL, eval, exec, command injection |
| XSS (A01) | innerHTML, dangerouslySetInnerHTML |
| Crypto (A02) | MD5, SHA1, Math.random |
| Config (A05) | CORS *, DEBUG mode |
| SSRF (A10) | URL concatenation |

### File Filtering
- Skips test files
- Skips node_modules
- Max file size: 40KB
- Max files: 40

### Context Extraction
- Line number
- Code snippet (±80 chars)
- File path
- Category & CWE mapping
